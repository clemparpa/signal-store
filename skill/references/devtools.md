# `@fluch/signal-store-devtools` — Redux DevTools bridge reference

Relays every `patchState` commit to the [Redux DevTools browser extension](https://github.com/reduxjs/redux-devtools/tree/main/extension). No Redux required — the extension is just the UI.

**Monitor-only**: action timeline, state tree, diff view. No time travel, no dispatcher, no skip/reorder (deliberately out of scope).

Peer dep: `@fluch/signal-store`. Install as a **devDependency**.

## `connectDevtools(store, options?)`

```ts
type ConnectDevtoolsOptions = {
  name?: string;        // default: 'signal-store'
  instanceId?: string;  // default: auto-generated by the extension
  maxAge?: number;      // default: 50 — how many actions the extension retains
  trace?: boolean;      // default: true — derive action name from stack
};

type DevtoolsConnection = { disconnect(): void };

connectDevtools(store: object, options?: ConnectDevtoolsOptions): DevtoolsConnection
```

## Quick start

```ts
import { signalStore, withState, withMethods, patchState } from '@fluch/signal-store';
import { connectDevtools } from '@fluch/signal-store-devtools';

const counter = signalStore(
  withState({ count: 0 }),
  withMethods((s) => ({
    increment: () => patchState(s, { count: s.count.value + 1 }),
    reset: () => patchState(s, { count: 0 }),
  })),
);

if (import.meta.env.DEV) {
  connectDevtools(counter, { name: 'Counter' });
}

counter.increment(); // DevTools: action "increment", state { count: 1 }
counter.reset();    // DevTools: action "reset",     state { count: 0 }
```

**Always wrap the call in a dev guard** — `import.meta.env.DEV` (Vite), `process.env.NODE_ENV !== 'production'` (webpack/Next), etc. The package is `sideEffects: false`, so the entire import tree-shakes out of production.

## Action naming

Default (`trace: true`): the connector parses `new Error().stack` at each commit, skipping internal frames (signal-store internals, RxJS, `node_modules`), and uses the first eligible function name as the action label.

```ts
withMethods((s) => ({
  addTodo: (text: string) => patchState(s, (state) => ({ todos: [...state.todos, text] })),
}));

store.addTodo('walk dog'); // DevTools action label: "addTodo"
```

Stack formats supported:
- V8 (Chrome/Node): `at fn (file:line:col)`
- SpiderMonkey/JSCore (Firefox/Safari): `fn@file:line:col`

Fallback when no eligible frame is found: `STATE_UPDATE`.

### Production / minified bundles

In minified production bundles, function names become mangled identifiers (`a`, `b`, …). If you ship `connectDevtools` to prod (e.g. for a beta), pass `trace: false` so every action is labelled `STATE_UPDATE`:

```ts
connectDevtools(store, { trace: false });
```

Better: don't ship it at all — the dev guard pattern is the recommendation.

## Multiple stores

Each `connectDevtools` call produces an independent connection. Pass distinct `name` values:

```ts
if (import.meta.env.DEV) {
  connectDevtools(authStore, { name: 'Auth' });
  connectDevtools(todosStore, { name: 'Todos' });
}
```

The extension's instance picker shows each by name.

## Lifecycle

The connector binds to the store's internal cleanup channel. When `destroyStore(store)` runs — explicitly, or automatically on React `<Provider>` unmount — the relay tears down. No memory leak, no manual handle to track.

For HMR or other cases where you want to disconnect without destroying the store:

```ts
const conn = connectDevtools(store);
// later…
conn.disconnect(); // store stays live; extension stops receiving actions
```

`disconnect()` is idempotent.

## No extension installed

Silent no-op — returns a stub `{ disconnect: () => {} }`. CI / headless environments don't crash on the import.

## What's not supported (on purpose)

- **Time travel** (`JUMP_TO_STATE` / `JUMP_TO_ACTION`): would need state snapshots or mutation replay. Out of scope without the dispatcher.
- **Dispatcher** (UI → store actions): would need an explicit action registry. The "no magic" principle argues against implicit method-name dispatch.
- **Skip / reorder**: requires a delta history (we keep snapshots only).

Open an issue if any of these become a blocker.

## Options reference

| Option | Default | Description |
|---|---|---|
| `name` | `'signal-store'` | Instance label in the DevTools picker. |
| `instanceId` | auto | Stable identifier; survives HMR reloads. |
| `maxAge` | `50` | Action retention in the extension. |
| `trace` | `true` | Derive action label from JS stack. `false` → all actions `STATE_UPDATE`. |

## Internal: `getMeta` via subpath

The connector reads `meta.state$` and `meta.mutations$` via `@fluch/signal-store/internal` (not part of the public API surface). You don't need to import from there directly.

## API

```ts
connectDevtools(store: object, options?: ConnectDevtoolsOptions): { disconnect(): void }
```

That's the entire surface.
